Stage 1.1: Basics of Web and HTML

Overview of the Web

Main types of document in the web is HyperText MarkupLanguage .
Computer-browsers make requests via internet to servers using http protocol.All types of files can be found on the web

Introduction to HTML

HTML can consists of :

1. Text content
2. Markup - Control appearances
3. References to other documents such as images, videos,etc
4. Links to other other webpages

Tags,Elements,Attributes
HTML documents made of elements which is expressed using tags. Some examples :
*Bold tags
*Italic tags
Elements can have attributes
Example of *HTML attributes such as link eg. Udacity

General Rules to Remember

Give literal instructions, use correct syntax, don't try to remember everything

Inline vs Block

inline - essentially put a command/elements in a line
block - make a box around a section which can be customized/stylized

Stage 1.2: Creating a Structured Document

Website Composition

Website consists of HTML (structure), CSS (style), Javascript (interactive components) and are governed by rules

Structure of a Website

To see a structure, use Chrome's Development Tools and "Inspect Element"

The structure of a website are "tree-like" or "branches"

To easily design a website, assume that websites are composed of multiple boxes. Each boxes can be given different styles and designs.

Text Editors (for programming)

Different tools for writing a webpages have varying strength and weaknesses. Scratchpad is easy to use, have instanteneous response time, but least powerful. Codepen is the middle ground. Sublime is most powerful but you have to save and preview on browser to see the result.

Stage 1.3: Adding CSS Style and HTML Structure

CSS

CSS = Cascading Style Sheets

Cascading means the most specific rules apply. Also can be explained using the concept inheritance. Basically this allows the website to be grouped into classes in which each classes have a specific style rules.

Avoiding repetition is important for consistency as well as to avoid mistakes. It also reduces burden on the processing times and server

Also, validate your works because different browsers might interpret things differently. This also helps to avoid mistakes.
